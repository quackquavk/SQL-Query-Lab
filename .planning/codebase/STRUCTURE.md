## Directory Purposes

**Root (`index.html`):**
- Single-page application shell
- Loads CDN dependencies (CodeMirror 5.65.16, sql.js 1.10.3)
- Loads `scripts/main.js` as ES module entry point
- Contains all HTML structure for the three-column app layout

**`scripts/`:**
- All application logic as ES modules
- 12 JS files, each with a focused responsibility
- Modules import from each other explicitly (no barrel files)

**`styles/`:**
- 8 CSS files organized by component/area
- `base.css` defines CSS custom properties (variables) for theming
- No CSS preprocessor — plain CSS with custom properties

**`.planning/codebase/`:**
- Generated by this analysis — not committed or part of app

## Key File Locations

**Entry Points:**
- `index.html` — HTML shell, CDN script loads, ` <script type="module">` for `main.js`
- `scripts/main.js` — `boot()` async function, UI wiring, all event handlers

**Configuration:**
- `index.html` — CSS and JS dependencies from CDN
- No build configuration, no bundler, no transpiler
- sql.js loaded from CDN as global script; CodeMirror loaded from CDN as global scripts

**Core Logic:**
- `scripts/runtime.js` — Mutable runtime singletons (SQL, editor, databases, cursor state)
- `scripts/state.js` — localStorage read/write with debounce
- `scripts/db.js` — sql.js wrapper: `cloneFromPristine()`, `activeDb()`, `persistSandboxDbDebounced()`
- `scripts/practice.js` — `loadQuestion()`, `runPracticeQuery()`, `compareResults()`
- `scripts/sandbox.js` — `setMode()`, `runSandboxQuery()`, `translateToMssql()`, snippet management

**Testing:**
- No test framework detected
- Manual testing via browser only

**Import Graph (key dependencies):**

- `main.js` imports from all other modules
- `db.js` imports `runtime.js` and `state.js` only
- `ui.js` imports `runtime.js`, `state.js`, `questions.js`, `db.js`, `utils.js`
- `sandbox.js` imports `runtime.js`, `state.js`, `db.js`, `ui.js`, `utils.js`, `practice.js`
- `practice.js` imports `runtime.js`, `state.js`, `db.js`, `ui.js`, `utils.js`, `format.js`
- `editor.js` imports `runtime.js`, `state.js`, `format.js`
- `runtime.js` is a pure export module — no imports
- `state.js` imports `runtime.js` (for `cursor`)
- `utils.js` and `format.js` are pure utility modules — no imports
- `questions.js` and `seeds.js` are pure data modules — no imports

## Naming Conventions

**Files:**
- kebab-case: `main.js`, `state.js`, `db.js`, `ui.js`, `format.js`, `utils.js`
- camelCase for compound: `sandbox.js`, `practice.js`, `editor.js`, `runtime.js`
- Data files: `questions.js`, `seeds.js`

**Functions:**
- camelCase: `runQuery()`, `setMode()`, `enterSandbox()`, `cloneFromPristine()`
- Verb-first pattern: `runPracticeQuery()`, `saveDraft()`, `persist()`, `markSolved()`

**Variables:**
- camelCase: `currentMode`, `currentDbName`, `currentQuestionId`
- PascalCase for classes: `SQL.Database` (sql.js)
- SCREAMING_SNAKE_CASE for constants: `MAX_HISTORY`, `STORAGE_KEY`

**Types (exported const arrays/objects):**
- PascalCase: `QUESTIONS`, `SEEDS`

## Where to Add New Code

**New Feature (e.g., new mode or major behavior):**
- Primary code: Add to `sandbox.js` if sandbox-related, `practice.js` if practice-related, or create new module
- Handler wiring: `main.js` `wireUI()` function
- State if needed: Add to `state.js` `defaultState()` and `loadState()`

**New Question:**
- Add entry to `scripts/questions.js` array
- Each question: `{ id, db, category, difficulty, title, prompt, hint, validationType, referenceQuery, verificationQuery, setupQuery }`

**New Database Seed:**
- Add SQL to `scripts/seeds.js` `SEEDS` object
- Add `<option>` to `index.html` `#dbSelect`

**New UI Rendering:**
- Add render function to `scripts/ui.js`
- Call from `main.js` wireUI or from hooks like `setUiHooks`

**Utilities:**
- Pure helpers → `scripts/utils.js`
- SQL formatting → `scripts/format.js`

## Special Directories

**CDN-loaded (not in repo):**
- CodeMirror 5.65.16 (JS + CSS)
- sql.js 1.10.3 (WebAssembly SQLite)

**No build output:**
- No `dist/`, `build/`, or compiled artifacts
- Directly served static files

**localStorage:**
- Key: `querylab:v1`
- Stores: solved questions, drafts, history, snippets, sandbox DB states, preferences
- Base64-encoded bytes for sandbox database blobs
