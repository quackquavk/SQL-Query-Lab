| Property | Value |

| Tool | none (vanilla CSS) |
| Preset | not applicable |
| Component library | none |
| Icon library | none (inline SVG or Unicode symbols) |

## Theme System

### Toggle Placement and Behavior
- **Location:** Top bar, right side, next to the existing menu button
- **Icon:** Sun (☀) for dark-to-light, Moon (☾) for light-to-dark
- **Size:** 32×32px touch target minimum, icon 18px
- **Tooltip:** "Switch to light theme" / "Switch to dark theme"
- **Transition:** 200ms ease on background-color, color, border-color properties only (no layout shifts)

### Implementation Approach
1. Add `data-theme` attribute to `<html>` element (`"dark"` or `"light"`)
2. CSS variable overrides in `[data-theme="light"]` selectors
3. Theme preference stored in `state.theme` (localStorage key `querylab:v1`)
4. On boot, apply saved theme or default to dark
5. Theme switch does NOT trigger full page reload

### System Preference Detection
- On first load, check `window.matchMedia('(prefers-color-scheme: light)')`
- If user has never set preference, use system preference as default
- Once user manually toggles, override system preference (stored in state)

### CSS Transition Strategy
Only animate color properties to avoid performance issues:
```css
*, *::before, *::after {
  transition: background-color 200ms ease, color 200ms ease, border-color 200ms ease;
}
/* Exceptions for elements that should not transition */
.CodeMirror, .CodeMirror * { transition: none; }
```

## Keyboard Shortcut System

### Existing Shortcuts (already implemented)

| Shortcut | Action | Location |
|----------|--------|-----------|
| `F5` | Run query | Editor |
| `Ctrl/Cmd + Enter` | Run query | Editor |
| `Ctrl/Cmd + Q` | Format SQL | Editor |
| `Ctrl/Cmd + Space` | Autocomplete | Editor |
| `Escape` | Close modal / cancel operation | Global |

### New Shortcuts for Phase 4

| Shortcut | Action | Context |
|----------|--------|---------|
| `Ctrl/Cmd + Shift + L` | Toggle light/dark theme | Global (works when editor not focused) |
| `Ctrl/Cmd + /` | Open keyboard shortcuts help modal | Global |
| `Ctrl/Cmd + Shift + N` | New tab | Tab bar focused |
| `Ctrl/Cmd + W` | Close current tab | Tab bar / editor |
| `Ctrl/Cmd + Tab` | Next tab | Global |
| `Ctrl/Cmd + Shift + Tab` | Previous tab | Global |
| `↑` / `↓` | Navigate autocomplete or menu | Editor autocomplete active |
| `Enter` | Confirm autocomplete / select menu item | Autocomplete or menu active |

### Keyboard Shortcut Discovery UI

**Shortcut help modal:**
- Triggered by: toolbar button with "⌘/Ctrl + /" label, or `Ctrl/Cmd + /` global shortcut
- Display: Modal with two-column layout (shortcuts left, context right)
- Grouped by context: Editor, Navigation, Query Execution, Theme & Display
- Shows macOS (⌘) and Windows/Linux (Ctrl) variants
- Each row: `[Icon] Action name — Shortcut display`
- Close on Escape or clicking backdrop

**Keyboard shortcut button in toolbar:**
- Location: Editor toolbar, right side, next to "Format" button
- Label: "⌘/" or keyboard icon
- Tooltip: "Keyboard shortcuts (⌘/)"
- ARIA: `aria-label="Keyboard shortcuts"`

### Implementation Notes
- All shortcuts stored in a `SHORTCUTS` constant object in `utils.js` or new `shortcuts.js`
- Global keyboard handler in `main.js` checks `runtime.cursor.currentMode` and `document.activeElement`
- Shortcuts disabled when CodeMirror is in certain states (selection active, etc.)
- No conflicts with browser native shortcuts (Ctrl+W, Ctrl+T, etc. are tab-specific and may need browser-side handling)

## Accessibility

### Skip Link (New)
**Location:** First element in `<body>`, visually hidden until focused
**Target:** `#main-content` (the `.app` element or `.center` section)
**Implementation:**
```html
<a href="#main-content" class="skip-link">Skip to main content</a>
```
```css
.skip-link {
  position: fixed;
  top: -100px;
  left: 16px;
  z-index: 9999;
  background: var(--accent);
  color: #1a1208;
  padding: 8px 16px;
  border-radius: 4px;
  font-weight: 600;
  font-size: 13px;
  text-decoration: none;
  transition: top 200ms ease;
}
.skip-link:focus {
  top: 16px;
  outline: 2px solid var(--accent-bright);
  outline-offset: 2px;
}
```

### Focus Management (New)
1. **Focus trap for modals:** When modal opens, focus moves to first focusable element inside modal. When modal closes, focus returns to element that triggered it.
2. **Focus visible indicators:** All interactive elements must have `outline: 2px solid var(--accent)` and `outline-offset: 2px` on `:focus-visible`. No `outline: none` without replacement.
3. **Roving tabindex for tab lists:** Mode toggle, results tabs, left tabs use roving tabindex (only active tab is in tab order).
4. **Editor focus:** When editor container receives focus, focus moves to CodeMirror instance.
5. **Announce dynamic changes:** Use `aria-live="polite"` region for feedback messages, query results count changes, and tab switches.

### ARIA Labels (Enhancements needed)

| Element | Current ARIA | Phase 4 ARIA |
|---------|--------------|--------------|
| Mode toggle | `role="tablist" aria-label="Mode"` | Keep existing, add `aria-controls` to each button |

| DB select | None | `aria-label="Select database"` |
| Editor container | None | `aria-label="SQL editor" aria-multiline="true"` |
| Results tabs | None | `role="tablist" aria-label="Results view"` |
| Left tabs | None | `role="tablist" aria-label="Left panel tabs"` |
| Schema table | None | `aria-label="Database schema"` |
| Modal | None | `role="dialog" aria-modal="true" aria-labelledby="modal-title"` |
| Toast | None | `role="status" aria-live="polite"` |
| Theme toggle | None | `aria-label="Switch to light theme" aria-pressed="false"` |
| Shortcuts button | None | `aria-label="Keyboard shortcuts" aria-keyshortcuts="Control+/"` |
| Tab bar | None | `role="tablist" aria-label="Open tabs"` |

### Landmark Roles (Enhancements needed)
- `<header class="topbar">` → `role="banner"`
- `<aside class="left">` → `role="complementary"`
- `<section class="center">` → `role="main"` with `id="main-content"`
- `<aside class="right">` → `role="complementary"`
- `<div class="modal">` → already has dialog role via ARIA

### Color Contrast (New verification required)
After implementing light theme, verify:
- All text/bg combinations meet WCAG AA (4.5:1 for normal text, 3:1 for large text)
- Interactive element boundaries have sufficient contrast
- Focus indicators visible in both light and dark themes
- Status colors (success, error, info) are distinguishable for colorblind users (use icons or text in addition to color)

### Motion Preferences

```css
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```
