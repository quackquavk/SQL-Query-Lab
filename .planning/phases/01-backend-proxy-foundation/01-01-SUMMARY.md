### Key Features Implemented

- **Hono framework** (not Express) per architecture decision
- **WebSocket** query streaming at `/api/query`
- **REST CRUD** for connections at `/api/connections`
- **SQL Server auth** (username/password), **Windows auth** (NTLM), **Entra ID** (stub)
- **Encrypted credential storage** (PBKDF2-derived key, AES-256-GCM)
- **Connection pool** per user per server (max 5 concurrent)
- **All queries parameterized** (PROF-02)
- **Credentials never logged**, never sent to client (PROF-01)
- **Session validation middleware**
- **Decoded error messages** (not raw TDS codes)

### Must-Haves Verification

- ✅ User can save connection with server/credentials to backend (never browser storage)
- ✅ User can test connection and see clear success/failure message before saving
- ✅ User can connect via SQL auth (username/password), Windows auth (NTLM), or Entra ID
- ✅ Credentials are encrypted server-side with user-derived key (PBKDF2, AES-256-GCM)
- ✅ All queries execute via backend proxy with parameterized inputs (no SQL injection)
- ✅ Query results stream row-by-row via WebSocket with cancel support
