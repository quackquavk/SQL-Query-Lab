## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0b0c0a` (--bg) | Main background |
| Secondary (30%) | `#17181410` / `#1a1b17` (--surface-1) | Cards, sidebar panels |
| Accent (10%) | `#e08a3c` (--accent) | Interactive elements, highlights |
| Destructive | `#e56b5a` (--error) | Destructive actions only |

**Accent reserved for:**
- ER diagram: selected table node border, FK relationship lines, zoom controls hover
- Execution plan: cost gradient baseline (green start), operator cost indicator
- Table designer: column type dropdown focus, constraint badge active state
- SP editor: syntax highlight keywords, Execute button
- All interactive hover states (4px solid border)

**Destructive reserved for:**
- `ALTER TABLE DROP COLUMN` confirmation
- `DROP PROCEDURE` / `DROP TABLE` actions
- Any rollback/revert actions

**Color for execution plan cost severity (D-09):**
| Cost Level | Color |
|------------|-------|
| Low (0-30%) | `#a3c968` (--success green) |
| Medium (30-70%) | `#e08a3c` (--accent amber) |
| High (70-100%) | `#e56b5a` (--error red) |

## Component Inventory

### ER Diagram Panel (left sidebar)
- **Container:** SVG element inside `.left-content`, overflow hidden, pan/zoom via D3 zoom behavior
- **Node (table):** Rounded rectangle, 180px min-width, auto-height, --surface-1 fill, --border stroke
- **Node header:** Table name in JetBrains Mono 12px, --text color, --surface-2 background
- **Column row:** 24px height, column name (mono 11.5px) + data type (mono 10.5px --text-dim), PK badge in --accent
- **FK indicator:** Arrow line connecting source column to target table, --accent color at 60% opacity
- **Selected state:** 2px solid --accent border, slight drop shadow
- **Zoom controls:** Bottom-right of panel, +/- buttons, 32px touch targets
- **Minimap:** Optional, bottom-left, 120x80px thumbnail of full diagram

### ER Diagram Selected Table Panel (right sidebar)
- **Trigger:** Single click on ER node
- **Panel:** Slides in from right, 320px wide, replaces/snippets panel
- **Header:** Table name in Instrument Serif italic 24px
- **Column list:** Grid rows with name, type, constraints (PK/FK/NULL badges)
- **Double-click hint:** Small text `double-click to open designer`

### Table Designer Modal
- **Overlay:** .modal-backdrop with blur, centered modal 800px wide max
- **Header:** `Table Designer` in Instrument Serif 32px, close button
- **Form:** Two-column layout — left for columns grid, right for constraints and DDL preview
- **Column grid:** Editable rows with dropdown cells for type/constraints
- **DDL preview:** Sticky panel below form, JetBrains Mono 12px, syntax colored, updates live on form change
- **Execute button:** --accent background, `Execute DDL` label, disabled state when preview empty

### Execution Plan Panel
- **Container:** SVG inside `.results` panel or new tab, hierarchical top-down D3 layout
- **Operator node:** Rounded rectangle, --surface-2 fill, 140px min-width
- **Operator details:** Name in 12px bold, row count (estimated vs actual), cost %
- **Connection lines:** Orthogonal (dagre), --border-bright color, arrow heads
- **Cost color:** Per-cost-severity gradient (green → amber → red)
- **Tooltip on hover:** Full operator details, estimated rows, actual rows, I/O cost, CPU cost

### Stored Procedure Editor (right sidebar tab)
- **Tab:** `.right-tab[data-tab="sp"]` — label `SP Editor`, icon `⚡`
- **Parameter strip:** Above editor, chips for each @param with type badge
- **Editor area:** CodeMirror 5 instance, T-SQL syntax mode
- **GO divider:** Vertical dashed line with `GO` label at 11px
- **Execute button:** Bottom of panel, --accent, `Save Procedure`
- **Validation inline:** Underline in --error color on syntax error, hover tooltip with message

### Interactive States (all components)
| State | Visual treatment |
|-------|------------------|
| Hover | --accent border or background shift, 0.12s transition |
| Active/Pressed | --accent-bright background, slight scale(0.98) |
| Selected | 2px solid --accent border |
| Disabled | 40% opacity, cursor not-allowed |
| Loading | Spinner or skeleton in --text-dim |
| Error | --error border, error icon, message below |

## Interaction Contracts

### ER Diagram
- **Pan:** Mouse drag on empty SVG area, cursor: grab → grabbing
- **Zoom:** Mouse wheel, zoom controls (+/- buttons), range 25%–400%
- **Select table:** Single click on node → highlight node + open columns panel in right sidebar
- **Edit table:** Double-click on node → open table designer modal
- **Deselect:** Click empty area → clear selection, hide columns panel

### Table Designer
- **Open:** Double-click ER node OR right-click table → "Design table"
- **Live DDL:** On any form field change → regenerate DDL preview in <16ms (debounced)
- **Add column:** Click `+ Add column` button → append new row to grid
- **Remove column:** Click `×` on column row → show destructive confirmation if column is not new
- **Execute:** Click `Execute DDL` → send to backend, show success/error toast, close modal on success
- **Cancel:** Click close or backdrop → confirm if unsaved changes exist

### Execution Plan Viewer
- **Load:** Query with `SET SHOWPLAN_XML ON` returns XML → parse → render D3 graph
- **Pan/Zoom:** Same as ER diagram
- **Operator details:** Hover → tooltip with full details; click → expand in-place details panel
- **Cost filter:** Toggle to show only operators above X% cost threshold

### Stored Procedure Editor
- **Load SP:** Click SP in object explorer → populate editor + extract parameters to strip
- **GO batches:** Editor shows visual `GO` divider between batches
- **Syntax validate:** On blur or after 800ms idle → send to backend validation endpoint → show inline errors
- **Save:** Click `Save Procedure` → backend CREATE/ALTER → success toast or error display
- **New SP:** Click `+ New` → empty editor, placeholder name `NewProcedure`
