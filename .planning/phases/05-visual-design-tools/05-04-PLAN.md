### Schema API:
- Query INFORMATION_SCHEMA.TABLES for table list
- Query INFORMATION_SCHEMA.COLUMNS for column details
- Query INFORMATION_SCHEMA.TABLE_CONSTRAINTS for PK/FK
- Query FOREIGN_KEY_COLUMN_USAGE for FK references

### Execution plan flow:
1. Execute `SET SHOWPLAN_XML ON` + user query
2. Capture the XML result (not rows)
3. Return the XML

### DDL execution:
1. Execute user DDL (CREATE/ALTER TABLE)
2. Return success or error message

### T-SQL validation:
- Uses `node-sql-parser` with T-SQL dialect
- Returns parse errors with line/column positions

### node-sql-parser usage (from RESEARCH.md):
```javascript
const { Parser } = require('node-sql-parser');
const parser = new Parser();

// Validate T-SQL
try {
  parser.parse(sql, { database: 'Tsql' });
  return { valid: true };
} catch (e) {
  return { valid: false, errors: [{ message: e.message, line: e.line, col: e.col }] };
}
```

       ```javascript
       // Enable showplan
       await pool.query('SET SHOWPLAN_XML ON');
       // Execute query - result is XML as single column
       const result = await pool.query(query);
       // Turn off showplan
       await pool.query('SET SHOWPLAN_XML OFF');
       // result.recordset[0] has XML string column (name varies by driver)
       const xml = result.recordset[0][Object.keys(result.recordset[0])[0]];
       return { xml };
       ```
