**`runLiveQuery(sql, options)`** — executes query via WebSocket:
- `options = { connectionId, timeout: 30000, pageSize: 100, page: 1 }`
- Reads `runtime.cursor.connectionId` — if absent, show feedback error
- Calls `apiClient.executeQuery(sql, connectionId, { timeout, connectionId })`
- Manages streaming rows:
  - On `columns`: initializes results view via `ui.renderResultsStreaming(columns, { pageSize, onPageChange, onSort, onExportCsv, onExportJson })`
  - On `rows`: calls `resultsView.addRows(rows)` (called incrementally per D-06)
  - On `done`: calls `resultsView.complete(executionTime, rowCount)`, shows execution time + row count via `ui.showFeedback('success', ...)` (per D-09, D-10)
  - On `error`: calls `resultsView.error(message)`, shows `ui.showFeedback('error', message)` (per D-11, EXEC-05)
- Stores `resultsView` reference in `runtime.cursor.currentResultsView`
- Sets `runtime.cursor.queryState = 'running'` then `'done'` or `'error'`
- Returns the results view handle

**`cancelLiveQuery()`** — cancels running query:
- If `runtime.cursor.queryState !== 'running'`, no-op
- Calls `apiClient.cancelQuery(runtime.cursor.currentQueryId)`
- Calls `resultsView.error('Query cancelled')`
- Resets `queryState` to `'cancelled'`

**`setMode('live')`** — switches to live mode:
- Sets `runtime.cursor.currentMode = 'live'`
- Sets `document.body.dataset.mode = 'live'`
- Adds CSS class `.mode-live` to body
- Hides sandbox-only UI elements, shows live-mode indicators
- If `!runtime.cursor.connectionId`, show connection required feedback
- Reuses existing CSS class pattern from sandbox/practice modes

**`enterLiveMode()`** — called by setMode('live') when connection is active:
- Shows live mode status bar: connection name, server name
- Enables editor keyboard shortcuts for live execution
- Loads live preferences from state.js (pageSize, timeout)
