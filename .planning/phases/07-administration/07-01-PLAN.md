```
SQL Agent (root node, always expanded)
 └─ Category: [category_name]
    └─ Job: [job_name] [status_icon]
```

- Green circle (`●`) — Enabled, last run succeeded
- Yellow circle (`●`) — Enabled, currently running
- Red circle (`●`) — Enabled, last run failed
- Gray circle (`○`) — Disabled

- Overview: enabled (yes/no), last run date, next run date, owner, description
- Steps: list of steps with name, type (Transact-SQL, CMDEXEC, etc.), outcome
- Schedules: schedule name, frequency (daily, weekly, etc.), next run time
- History: paginated table — Run Date, Duration, Status, Message
- Alerts: list of linked alert names with badge count

- `Start` → POST `/api/sql-agent/job/:db/:name/start`

- `Disable` → POST `/api/sql-agent/job/:db/:name/disable`

- `Properties` → switches to Overview tab

- Fetch jobs: `apiClient.fetchSqlAgentJobs()`

- Fetch job history: `apiClient.fetchJobHistory(jobName, page)`
- Job actions: `apiClient.startJob()`, `apiClient.stopJob()`, `apiClient.enableJob()`, `apiClient.disableJob()`

Use the hook-injection pattern. Receive callbacks via `setJobBrowserHooks({ showFeedback, switchTab })`.

Follow existing codebase conventions: 2-space indent, single quotes, template literals for HTML, ES modules with named exports only.

Step 2 — Point-in-Time (optional):

- If checked: datetime picker appears

- Database name input (pre-filled with current name)
- Checkbox: "Overwrite existing database" (requires typing name to confirm)

Step 4 — Confirm:
- Shows full restore command that will be executed
- Text input requiring typing target database name to enable "Restore" button
- "Copy command" button

**Progress Bar (WebSocket):**


<name>Task 4: Add backend backup/restore routes</name>
<files>backend/routes/backupRestore.js</files>
<action>
Create `backend/routes/backupRestore.js` with Hono route handlers:

**Endpoints:**

1. `POST /api/backup` — Execute backup with WebSocket progress
   ```javascript
   // Body: { dbName, backupType: 'full'|'diff'|'log', destination, compression, checksum, backupSetName }
   // Opens WebSocket connection to stream progress
   // Uses BACKUP DATABASE/LOG T-SQL command
   // Progress events every 10% or every 5 seconds
   // Returns final result { success, backupSetId, verified }
   ```

2. `GET /api/backup/history/:db` — List backup history
   ```javascript
   // Returns: { backups: [{ date, type, size, firstLsn, lastLsn, deviceType }] }
   // Query: msdb.dbo.backupset JOIN dbo.backupmediafamily
   ```

3. `POST /api/restore` — Execute restore with WebSocket progress
   ```javascript
   // Body: { dbName, backupPaths: [...], pointInTime, overwrite }
   // Multi-database restore (full + diff + logs in sequence)
   // WebSocket progress streaming
   // Returns final result { success }
   ```

4. `POST /api/restore/verify` — Verify backup file
   ```javascript
   // Body: { backupPath }
   // Runs RESTORE VERIFYONLY
   // Returns: { valid: true/false, message }
   ```

**WebSocket Progress Event Format:**
```javascript
// Progress
ws.send(JSON.stringify({ type: 'progress', percent: 50, currentFile: 'db.bak', elapsed: 1500 }))

// Complete
ws.send(JSON.stringify({ type: 'complete', success: true, backupSetId: 'xyz' }))

// Error
ws.send(JSON.stringify({ type: 'error', message: 'Disk full', suggestion: 'Free space or enable compression' }))
```

**Progress Streaming Implementation:**
- Query executes with `STATISTICS ON` for byte counts
- Calculate percent from `total_bytes` vs `forwarded_bytes`
- Stream progress every 10% change or 5 seconds (debounced)
- WebSocket connection passed to service layer

**Error Handling:**
- Database in use → 409 with "Cannot restore: database in use by other connections"
- Insufficient disk space → 507 with suggestion to enable compression
- Corrupt backup → 422 with "Backup file is corrupt"

Follow Hono patterns. Use existing `sqlServer.js` pool. WebSocket upgrade using `ws` library pattern from Phase 2.
</action>

```javascript
// List all jobs
export async function fetchSqlAgentJobs() {
  const res = await fetch(`/api/sql-agent/jobs/${currentDb}`)
  return res.json()
}

// Get job details (steps, schedules, alerts)
export async function fetchJobDetails(jobName) {
  const res = await fetch(`/api/sql-agent/job/${currentDb}/${encodeURIComponent(jobName)}`)
  return res.json()
}

// Get job history (paginated)
export async function fetchJobHistory(jobName, page = 0) {
  const res = await fetch(`/api/sql-agent/job/${currentDb}/${encodeURIComponent(jobName)}/history?page=${page}&pageSize=50`)
  return res.json()
}

// Job control actions
export async function startJob(jobName) {
  const res = await fetch(`/api/sql-agent/job/${currentDb}/${encodeURIComponent(jobName)}/start`, { method: 'POST' })
  return res.json()
}

export async function stopJob(jobName) {
  const res = await fetch(`/api/sql-agent/job/${currentDb}/${encodeURIComponent(jobName)}/stop`, { method: 'POST' })
  return res.json()
}

export async function enableJob(jobName) {
  const res = await fetch(`/api/sql-agent/job/${currentDb}/${encodeURIComponent(jobName)}/enable`, { method: 'POST' })
  return res.json()
}

export async function disableJob(jobName) {
  const res = await fetch(`/api/sql-agent/job/${currentDb}/${encodeURIComponent(jobName)}/disable`, { method: 'POST' })
  return res.json()
}
```

**Backup/Restore:**
```javascript
// Execute backup (returns promise, WebSocket for progress)
export async function executeBackup(options) {
  const res = await fetch('/api/backup', {
    method: 'POST',

  })
  return res.json()
}

// List backup history

// Execute restore
export async function executeRestore(options) {
  const res = await fetch('/api/restore', {
    method: 'POST',

    body: JSON.stringify(options)
  })
  return res.json()
}

// Verify backup

    method: 'POST',
    headers: { 'Content-Type': 'application/json' },

}
```
