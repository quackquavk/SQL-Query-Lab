  <name>Task 2: Add auto theme detection via prefers-color-scheme</name>
  <files>styles/base.css</files>
  <read_first>styles/base.css</read_first>
  <action>
  Add `@media (prefers-color-scheme: light)` block inside the `[data-theme="auto"]` selector. Place it after the `[data-theme="dark"]` block and before the `[data-theme="light"]` block or as a nested rule within auto. The auto block should:

  1. Read system preference: `window.matchMedia('(prefers-color-scheme: dark)')`
  2. Apply dark or light theme accordingly
  3. Listen for changes via `change` event

  Actually, the CSS-only approach is preferred: use `@media (prefers-color-scheme: dark)` inside `[data-theme="auto"]`:

  ```css
  [data-theme="auto"] {
    /* default: dark (matches unspecified) */
  }

  @media (prefers-color-scheme: light) {
    [data-theme="auto"] {
      --bg: #f5f3ef;
      --bg-elev: #ffffff;
      /* ... all light values ... */
    }
  }
  ```

  This applies light values only when system prefers light AND data-theme is auto. The JS only needs to set initial `data-theme` attribute on `<html>`.
  </action>
